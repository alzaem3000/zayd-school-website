# تقييم كود المشروع (مراجعة سريعة)

## الملخص التنفيذي
المشروع يحتوي على أساس جيد (React + Vite + TypeScript + Vitest)، لكن حالة المستودع الحالية فيها **مشكلة بنيوية عالية التأثير**: ملفات الإعداد تشير إلى هيكل مجلدات (`client/`, `server/`, `shared/`) غير موجود بالشكل المتوقع داخل النسخة الحالية، وبالتالي أوامر الفحص والبناء والاختبار لا تعمل بشكل طبيعي.

## ما تم التحقق منه
- `npm run check` يفشل لأن `tsconfig.json` يبحث عن ملفات TypeScript داخل مسارات غير موجودة.
- `npm test` يفشل لأن `vitest.config.ts` يبحث عن الاختبارات داخل `tests/` فقط.
- `npm run build` يفشل لأن `vite.config.ts` يحدد `root` على `client` ويتوقع وجود `client/index.html`.

## ملاحظات تقنية حسب الأولوية

### 1) خلل هيكل المشروع مقابل ملفات الإعداد (أولوية حرجة)
- `tsconfig.json` يستخدم `include` لمسارات `client/src/**/*`, `shared/**/*`, `server/**/*`.
- `vite.config.ts` يحدد `root` على مجلد `client` ويضبط alias `@` على `client/src`.
- هذا يسبب فشل البناء المباشر (`Could not resolve entry module "client/index.html"`).

**الأثر:** لا يمكن الاعتماد على CI/CD أو أي فحص جودة لأن الأدوات الأساسية لا تعمل.

### 2) إعداد الاختبارات لا يلتقط الاختبارات الفعلية (أولوية عالية)
- `vitest.config.ts` يحدد `include` على `tests/**/*` فقط.
- يوجد ملف اختبار في الجذر (`logic.test.ts`) لكنه لا يُلتقط.

**الأثر:** نتيجة الاختبارات تعطي انطباعًا مضللًا بأن المشروع بلا اختبارات.

### 3) `.gitignore` غير مناسب لنوع المشروع الحالي (أولوية عالية)
- `.gitignore` الحالي خاص بقوالب AL (Dynamics 365) ولا يتضمن تجاهلًا قياسيًا لـ Node.js/TypeScript (`node_modules`, `dist`, `*.log`, ...).

**الأثر:** احتمالية إدخال ملفات غير مطلوبة في Git ورفع حجم المستودع بشكل كبير.

### 4) اتساق مسارات الاستيراد يحتاج إعادة ضبط (أولوية متوسطة)
- `App.tsx` يعتمد مسارات alias مثل `@/components/...`, `@/pages/...`, `@/hooks/...`.
- هذا يتطلب بنية مجلدات متسقة مع إعدادات alias، وهي غير واضحة في الحالة الحالية.

**الأثر:** صعوبة الصيانة، وتكرار أخطاء الاستيراد/البناء.

## خطة إصلاح مقترحة (عملية وقصيرة)
1. **تثبيت هيكل المجلدات أولًا**: إما
   - (أ) إعادة الملفات إلى البنية الأصلية (`client/src`, `server`, `shared`)، أو
   - (ب) تعديل `vite.config.ts` و`tsconfig.json` وalias لتطابق البنية الحالية.
2. **تصحيح إعدادات الاختبار**:
   - توسيع `include` في Vitest ليلتقط `*.test.ts` في الجذر أو جميع المجلدات.
3. **استبدال `.gitignore`** بقالب Node/Vite/TypeScript مناسب.
4. **إضافة بوابة جودة بسيطة**:
   - نجاح `npm run check` + `npm test` + `npm run build` قبل أي merge.
